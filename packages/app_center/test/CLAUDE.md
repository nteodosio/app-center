# Unit Tests

All test files live flat in this directory ‚Äî do not create subdirectories mirroring `lib/`.

## Running

```bash
cd packages/app_center
fvm flutter test                          # All tests
fvm flutter test test/snap_page_test.dart # Single file
fvm flutter test --name "test name"       # By name
```

## Test Utilities

@test_utils.dart provides:
- `pumpApp(builder)` ‚Äî pumps a `MaterialApp` with l10n, theming, and Ubuntu font
- `createContainer(overrides)` ‚Äî creates a `ProviderContainer` with auto-dispose and `MemoryFileSystem`

## Data Builders

Use factory functions from @test_utils.dart to create test data with sensible defaults:
- `createSnap(...)` ‚Äî builds a `Snap` with optional overrides
- `createAppstreamComponent(...)` ‚Äî builds an `AppstreamComponent`
- `createLocalDebInfo(...)` ‚Äî builds a `LocalDebInfo`
- Pre-built constants: `defaultInstalledDeb`, `defaultDebWithUpdate`

## Mocks

Generated by Mockito via `@GenerateNiceMocks` in @test_utils.dart. After adding/changing mock annotations, run `melos generate`. Do not edit `*.mocks.dart` files.

Two kinds of mock helpers:
- `registerMock*Service()` (e.g. `registerMockSnapdService()`) ‚Äî creates and registers a mock in the `ubuntu_service` registry. Use for services the code under test fetches via `getService`.
- `createMock*()` (e.g. `createMockPackageKitTransaction()`) ‚Äî returns a mock without registering. Use when passing directly as a parameter.

Search-specific: `createMockSnapSearchProvider(Map)` / `createMockDebSearchProvider(Map)` take a map of query‚Üíresults.

## Teardown

Every test file must include `tearDown(resetAllServices)` to clean up registered mock services between tests.

## Patterns

**Widget tests:** register mock services in `setUp`, pump the widget with `pumpApp`, assert with `find.*` matchers. Uses `yaru_test` and `ubuntu_test` for Yaru-specific finders (e.g. `find.button()`, `find.textField()`). Access l10n strings via `tester.l10n`.

**Provider/model tests:** create a container with `createContainer(overrides: [...])`, read providers with `container.read(provider.future)`, assert on returned data.
